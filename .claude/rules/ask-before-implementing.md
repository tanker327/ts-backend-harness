# Ask Before Implementing

## Rule
When the user raises ANY improvement, fix, or new feature — ALWAYS ask first:
> "Should I implement this now, or add it to progress tasks for later?"

Do NOT start writing code, creating files, or making changes until the user explicitly says to proceed.

## Why
The user controls prioritization. Not every request means "do it right now." It might be better tracked in progress/current.json and done in priority order.

## When this applies
- New feature requests
- Bug fixes (unless they block the current task)
- Refactoring suggestions
- Any code change the user describes or asks for

## Exceptions
- Blocking issues during an already-approved task — fix inline and note in the task
- The user explicitly says "do it now" or "implement this" in the same message
- Trivial clarification questions that don't involve code changes
